QueryPath View more...

This is helpful for your multiple-layer project developing. There are two kind QueryPaths:

  • QueryPath It allows you to specify a topology graph with to unlimited breadth and depth to fetch the related ORM entities of the current center ORM entity very easy and safe
  • SimpleOrderPath it allows you to specify a topology graph with unlimited breadth and depth to order the current ORM entity and related ORM entities

QueryPath has no depenency with any ORM technologies so that it can be used by the components of any layer, for example, both data accessing component and business component. You can use it to be the parameter of the query methods of the components of any layer so that the UI layer can specify the QueryPath and assign it to data accessing layer through several middle layers. Let’s say goodbye to Spring’s “open session in view” and N+1 query.

BabyFish enhanced both Hibernate API and JPA, for both Hibernate and JPA progject, you can use it.

DistinctLimitQuery View more...

In Hibernate, if you use paging query with collection fetches, hibernate will query all the data from database and apply the paging in memory. The DistinctLimitQuery of babyfish can generate the paging SQL for paging query with collection fetches when the database is Oracle.

ObjectModel View more...

ObjectModel is the most important functionality of BabyFish, it is the root cause why I created BabyFish.

This is helpful for your data model class design. Often, we design 2 model classes, each one use an association property to retain other one, that is called bidirectional association. If you do nothing anymore, When your program changes the association by one side, but the other side will not be changed correspondingly and automatically, ObjectModel can help you to resolve this problem by annotation declaration, not by code! Babyfish supports two kinds of ObjectModel

  • ObjectModel4Java: ObjectModel can be used for any java data model class in any java application, it is not only confined in the ORM entity classes.
  • ObjectModel4JPA: You can use it in your JPA/Hibernate entity, it can synchronize the bidirectional association and it can work with ORM's lazy-loading collection together.
