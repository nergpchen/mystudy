Requirements

Component Optional Version Download URL Remark
JDK Required >=7 http://www.oracle.com/technetwork/java/javase/downloads/index.html Babyfish does not support Java6 or earlier
Apache Maven Required >=3.1 http://maven.apache.org/download.cgi# Don't forget add the directory of "mvn" command into the PATH variable of OS
Oracle Server Optional >=11g http://www.oracle.com/technetwork/database/enterprise-edition/downloads/index.html Oracle is optional, but it would better be downloaded and installed, because babyfish supports a functionality named DistincitLimitQuery that can markedly optimize the peformance when the database is Oracle

Variable Assumption

  • Let's call the directory of the unziped directory of babyfish(the directory of this page) ${babyfish-dir}
  • Let's call the oracle host name ${oracle-host}. It should be localhost(or 127.0.0.1) if the oracle is installed on the same machine.
  • Let's call the oracle port number ${oracle-port}. Its default value should be 1521.
  • Let's call the oracle instance name ${oracle-sid}. It's decided by the user when the Oracle Server is installed.

Attention please

Before taste this framework, please pay attention to the following two points.

  1. In the directory "${babyfish-dir}/install", there are many "bat" files and "sh" files. All of them should be executed under its own directory.
    Illegal example: 
    ${babyfish-dir}/install/babyfish/install.sh
    Concurrent example: 
    cd ${babyfish-dir}/install/babyfish
./install.sh
  2. Though the shell file "${babyfish-dir}/install/ojdbc/install.sh" will not be executed by the user directly; but it's depended by other shell files. In order to make other shell files can be executed successfully, plase execute 
    chmod +x ${babyfish-dir}/install/ojdbc/install.sh
    before execute other shells files if the current linux account does not have the execution permission of that file.

Install the babyfish framework

You can also skip the "babyfish framework installation" and do Run Demo directly, because babyfish has been uploaded to Maven Central Repository on 2015-10-23

There are two ways to test the babyfish framework

  1. Use the install script 
    cd ${babyfish-dir}/install/babyfish
./install.sh(For windows: it should'be "install.bat")
<Finally, please answer all the questions>
  2. Use maven directly

    During the babyfish framework installation, there are two ways to test all the functionalities.

    1. Use hsqldb memory database 
      cd ${babyfish-dir}/src
mvn clean install
    2. Use the oracle database 
      cd ${babyfish-dir}/src
mvn clean install -Doracle=jdbc:oracle:thin:@${oracle-host}:${oracle-port}:${oracle-sid} -Doracle.user=<your_user> -Doracle.password=<your_password>

Run demo

Part-I: Compile and test all the demos

There are two ways to install the babyfish demo

  1. Use the install script 
    cd ${babyfish-dir}/install/babyfish
    ./install-demo.sh(For windows: it should'be "install-demo.bat")
    <Finally, please answer all the questions>
  2. Use maven directly
    1. Use hsqldb memory database 
      cd ${babyfish-dir}/demo
mvn clean install
    2. Use the oracle database 
      cd ${babyfish-dir}/demo
mvn clean install -Doracle=jdbc:oracle:thin:@${oracle-host}:${oracle-port}:${oracle-sid} -Doracle.user=<your_user> -Doracle.password=<your_password>
Note:

In order to execute the test cases about Oracle by Eclipse IDE, the generated eclipse project file contains ojdbc dependency. There are two ways to make the generated eclipse project files contain that.

  1. Choose Oracle during answer questions of install-demo.sh(or install-demo.bat)
  2. cd ${babyfish-dir}/demo
mvn eclipse:eclipse -Doracle

In Eclipse IDE, when open "Run Configrations..."(or "Debug Configurations...") dialog to create the junit runner, please add "-Doracle=jdbc:oracle:thin:@${oracle-host}:${oracle-port}:${oracle-sid} -Doracle.user=<your_user> -Doracle.password=<your_password>" into the text field "VM arguments" of the tab page "Arguments".

Maybe sometimes some test cases can't be passed under Eclipse JUnit runner, that's because some projects use the maven plugin(ant task) to do some byte-code instrument work in compilation-time so that the final byte-code is not the simple compilation result of java soruce file but Eclipse may do some default compliation when these projects are imported. If this problem happen, use maven to recompile the project that reports error, then the test cases can be passed by Eclipse IDE.

If demo is installed sucessfully, please view Demo Learning Guide to learn the functionality of babyfish framework one by one.

Part-II: Run the comprehensive demo: war3shop

Babyfish-Demo contains an integrative web project: "war3shop", its directory is "${babyfish-dir}/demo/babyfishdemo-war3shop". Now let's run it.

Note:

Be different with this comprehensive demo that shows all the functionalities of babyfish framework together, the other demos show the functionalities one by one, please view Demo Learning Guide to learn other demos one by one.

Step-I: Start the database server

There are two ways to run war3shop, standalone hsqldb server or Oracle.

Oracle is suggested because babyfish supports a functionality named DistincitLimitQuery that can markedly optimize the peformance when the database is Oracle, but hsqldb standalone server is another choice if there is no Oracle server.

  1. In order to start the hsqldb server, please use this script: 
    cd ${babyfish-dir}/install/war3shop-demo
./start-hsqldb.sh(For windows: it should'be "start-hsqldb.bat")

    Press "Ctrl+C" when it should be stopped.

  2. Orace is suggested, please make sure the Oracle instance is started.

Step-III: Start the war3shop web server

The previous step can generate a new file "start-war3shop-by-oracle.sh" (or "start-war3shop-by-oracle.bat", "start-war3shop-by-hsqdb.sh", "start-war3shop-by-oracle.bat"). Please run this generated file to start the jetty server.

    cd ${babyfish-dir}/install/war3shop-demo
./start-war3shop-by-oracle.sh(For windows: it should'be "start-war3shop-by-oracle.bat")
  • If hsqldb is specified in previous step 
    cd ${babyfish-dir}/install/war3shop-demo

press "Ctrl+C" when it should be stopped.
