BabyFish: To .NET Developers
Introduction Get Started Demo Guide To .NET Developers

Thank .NET framework for providing the inspiration for babyfish, This document is written for the devlopers who have experience of .NET framework, it lists some analogies between the types of babyfish and the types of .NET framework.

Note:
In this table, the text regions of some class names are gray and italic, that means those classes are internal so that they should not be used by the user directly.

org.babyfish.model.metadata.ObjectModelDeclaration System.Data.Objects.DataClasses.EdmEntityTypeAttribute
  • [EdmEntityTypeAttribute] or [EdmComplexTypeAttribute] is used by the model class; but @ObjectModelDeclaration is used by the nested object model interface of model class.
  • [EdmEntityTypeAttribute] or [EdmComplexTypeAttribute] can only be used by ORM model class; but @ObjectModelDeclaration can be used by both simple model class and ORM model class.
System.Data.Objects.DataClasses.EdmComplexTypeAttribute
org.babyfish.model.metadata.Scalar System.Data.Objects.DataClasses.EdmScalarPropertyAttribute
  • [EdmScalarPropertyAttribute] is used by properties of the model class; but @Scalar is used by properties of the nested object model interface of model class.
  • [EdmScalarPropertyAttribute] can only be used by ORM model class; but @Scalar can be used by both simple model class and ORM model class.
org.babyfish.model.metadata.Association System.Data.Objects.DataClasses.EdmRelationshipAttribute
  • [EdmRelationshipAttribute] is used by properties of the model class; but @Association is used by properties of the nested object model interface of model class.
  • [EdmRelationshipAttribute] can only be used by ORM model class; but @Association can be used by both simple model class and ORM model class.

org.babyfish.reference.Reference<T> System.Data.Objects.DataClasses.EntityReference<TEntity>
  • Not only Reference, but also IndexReference which is opposition of list and KeyedReference which is opposition of map is supported by BabyFish.
  • Reference association properites are supported by BabyFish for both ORM model class and simple mode class.

org.babyfish.persistence.XQuery.setQueryPaths(org.babyfish.persistence.path.QueryPath ... queryPaths) System.Data.Objects.ObjectQuery<T>.Include(String path)
  • "ObjectQuery<T>.Include(String path)" can only accept the string which is untyped so that errors of path can only be reported at runtime; "XQuery.setQueryPath(QueryPath ... queryPaths)" accepts QueryPath which can be typed path so that the errors of path can be reported at compilation-time.
  • "ObjectQuery<T>.Include(String path)" only supports IncludePath(FetchPath), "XQuery.setQueryPath(QueryPath ... queryPaths)" supports both FetchPath and SimpleOrderPath.
